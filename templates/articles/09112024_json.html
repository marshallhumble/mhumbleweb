Upgrade a Running Docker Container


I was looking for a way to upgrade a Docker image in a running container and had to do some digging to find it. So, I thought I would pass the knowledge on. This method also works for upgrading a database container while preserving its data.


Overall, the process is straightforward. First, ensure you are in the same directory as your Docker Compose file. Then, run the following command: 


    
      ❯❯❯  docker compose pull && docker compose up -d && docker image prune -af

        [+] Pulling 12/12
        ✔ adminer Pulled                                         0.9s
        ✔ db Pulled                                              9.7s
        ✔ ed6a7145c00e Pull complete                             3.2s
        ...
        ✔ 0cc6c2d50cf6 Pull complete                             8.3s

In my setup, I am running Adminer and MySQL in this Docker container. The command docker compose pull fetches the latest image versions. The docker compose up command, as noted in the Docker documentation, "Builds, (re)creates, starts, and attaches to containers for a service." The -d flag runs the container in detached mode, allowing it to continue running in the background.


The final command, docker image prune -af, cleans up any unused images that do not have at least one container associated with them. The -a flag removes all unused images, and -f forces deletion without confirmation. Running this regularly helps free up disk space, as unused images can accumulate quickly. 


        
        [+] Running 2/2
        ✔ Container webfileshare-adminer-1  Running                    0.0s
        ✔ Container webfileshare-db-1       Starting                   1.9s
        Deleted Images:
        untagged: mysql@sha256:..
        deleted: ...

        Total reclaimed space: 609.1MB

Over time, these unused images can accumulate, so it’s good practice to prune them periodically to free up space.
